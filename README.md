# Conflicts Solving Agents — AI 辅助分歧解决平台

> 通过结构化的分组讨论与 AI 辅助梳理，帮助多方就有分歧的问题找到兼顾各方核心诉求的解决方案。

## 这是什么？

在团队协作、项目决策、方案评审等场景中，多方之间的分歧往往难以高效收敛。传统的讨论方式容易陷入无序争论，核心矛盾被淹没在大量细节中。

Conflicts Solving Agents 提供了一种结构化的分歧解决机制：

1. 各方围绕一个议题，分别阐述自己的立场、事实依据和论据
2. 各方独立提出自己的解决方案
3. AI 裁判自动梳理各方方案中的共识与矛盾
4. 各方针对矛盾点进行有针对性的讨论和反思
5. 经过多轮迭代，逐步消除分歧，最终收敛为一个兼顾各方合理诉求的方案

整个过程中，AI 不替代人做决定，而是作为中立的梳理者和组织者，帮助讨论保持结构化和高效。人工在每轮结束后介入确认，始终掌握最终决策权。

## 核心特性

- **多方参与**：支持两方或多方同时参与讨论，每方由独立的 AI Agent 代理，确保上下文隔离
- **结构化流程**：立论 → 提交方案 → AI 梳理 → 讨论挑战 → 反思修正 → 人工确认，循环迭代最多 5 轮
- **AI 裁判梳理**：自动提取各方方案中的共识与矛盾，生成综合方案，让讨论聚焦在真正的分歧点上
- **事实与论据管理**：各方的事实和论据在讨论过程中可被挑战、修正或放弃，所有变更留有完整记录
- **人工介入机制**：每轮结束后各方人工确认，可补充事实、调整论据，所有参与方确认后才进入下一轮
- **直接终论申请**：任一方可在人工确认阶段发起直接终论投票，达到多数同意即可提前结束
- **完整过程记录**：每轮的挑战、反思、变更、裁判判定均被持久化记录，支持完整复盘
- **多场并发隔离**：支持同时进行多场讨论，各场次数据和 Agent 实例完全隔离

## 适用场景

- 团队内部技术方案选型存在分歧
- 产品需求优先级排序各方意见不一
- 项目资源分配方案需要多方协商
- 规则制定或政策讨论需要兼顾多方利益
- 任何需要多方就同一问题达成共识的场景

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | FastAPI + Uvicorn |
| AI Agent | LangGraph (StateGraph + create_react_agent) + LangChain |
| Agent 持久化 | LangGraph SqliteSaver (Checkpointing) |
| 数据存储 | 本地文件系统 (JSON) |
| 实时推送 | SSE (Server-Sent Events) |
| 前端 | React 18 + TypeScript + Vite |
| 部署 | Docker Compose |

## 快速开始

### 环境要求

- Docker & Docker Compose
- 一个兼容 OpenAI API 的大语言模型服务（如 OpenAI、Azure OpenAI、或其他兼容服务）

### 启动服务

```bash
git clone https://github.com/your-username/conflicts-solving-agents.git
cd conflicts-solving-agents
docker compose up -d
```

服务启动后访问 `http://localhost:8080`。

### 配置 AI 模型

首次使用前，进入系统配置页，填写 AI 模型的 API URL 和 API Key。

### 开始一场讨论

1. 在议题列表页创建新议题，描述需要解决的分歧问题
2. 添加参与方（至少两方），每方填写自己的立场、事实和论据
3. 所有参与方完成立论后，启动讨论
4. AI 自动驱动讨论流程，各方在人工确认环节介入
5. 讨论收敛后，查看最终解决方案

### 本地开发

```bash
# 后端
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev
```
